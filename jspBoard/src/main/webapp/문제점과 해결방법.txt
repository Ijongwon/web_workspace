지금 구현한 부분까지의 문제점
1. 수정버튼은 로그인한 사용자의 글만 보이지만
url에서 http://localhost:8081/jspBoard/community/update.jsp?idx=50&page=1
의 idx를 로그인한 사용자의 글이 아닌 아무값이나 입력하면 수정이 가능함
또는 로그인을 하지않아도 아무글이나 실행되는지 확인

2. 로그인을 안하면 글쓰기에서 로그인하도록 하지만
url에서 http://localhost:8081/jspBoard/community/writeAction.jsp?title=테스트&context=zzz&writer=hongGD
를 로그인 안한 상태에서 직접 url 입력하고 실행하면 됨

*** 개선 : 글번호 표시 기본키값 대신 행번호(rownum)을 글 목록에 표시하는것을 추천
		  post 방식으로 바꿀것 update.jsp , delete.jsp
		  post 방식을 요청이 왔는지 확인하고 처리하기

해결방법 첫번째 : 항상 먼저 update.jsp, updateAction.jsp, writeAction.jsp 등 필요한 곳에서 session 검사 
      -> 로그인 세션 저장값을 비교해서 일치하지않은 exception 발생
      -> exception 또는 404 오류가 생겼을때 보일 페이지 error.jsp를 web.xml에 설정
      
해결방법 두번째 : 모든 필요한 소스파일에 new Exception() 처리하는것도 번거로움
      -> 웹컨테이너에는 필터(filter) : 요청에 대한 처리를 하기 전에 요청에 대한 검사와 사전 작업을 할 수 있음
      							 응답에 대한 처리를 하기 전에 응답에 대한 검사와 사전 작업을 할 수 있음
      
* 인증 : 로그인과 같이 사용자 확인 

* 인가 : 인증 후 권한. 예를들면 로그인 사용자가 작성한 글만 수정권한이 있음(인가)

SessionFilter 구현하기
: 메뉴로 클릭해서 사용자 기능을 사용하는 것이 정상이지만
어떤 사용자는 아무 의미 없이 장난으로 (또는 나쁜 의도로) url 에 직접 주소를 입력해서 실행할 수도 있기 때문
잘못된 요청을 받지 않도록 session 검사를 하기 위한 필터를 구현해 봅니다.
- 로그인 했을 때 접근할 수 없는 url은 index.jsp 로 리다이렉트
- 로그인 안 했을 때 접근 할 수 없는 url은 index.jsp 로 리다이렉트
